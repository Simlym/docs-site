# 核心特性

OpenClaw 围绕「统一、轻量、可扩展」三大原则设计，主要功能点如下。

## 🧰 统一工具入口

- **单一命令** `openclaw` 访问所有子工具，支持子命令嵌套与别名。
- **智能补全** —— 内置 Bash / Zsh / Fish / PowerShell 自动补全脚本。
- **统一帮助系统** —— 每条命令自带 `--help`，支持中文描述。

## ⚡ 轻量高效

- **零侵入** —— 不修改系统全局环境，所有文件集中在 `~/.openclaw/`。
- **懒加载** —— 工具按需加载，避免启动时的全量扫描开销。
- **快速启动** —— 冷启动在毫秒级完成。

## 🔌 插件与扩展机制

- **内置插件市场** —— 一条命令安装社区插件：`openclaw plugin add <name>`。
- **自定义工具** —— 将任意脚本放入 `tools/` 目录即自动注册为子命令。
- **生命周期钩子** —— 支持在命令执行前后注入自定义逻辑。

## 🔄 可分享的配置

- **配置即代码** —— 全部配置以 TOML 描述，可纳入 Git 版本管理。
- **配置继承** —— 支持团队基线配置 + 个人覆盖配置的分层合并。
- **一键同步** —— `openclaw sync` 在多台机器间同步工具与配置。

## 🖥️ 跨平台一致性

- **统一行为** —— 同一条命令在 Linux / macOS / Windows 上语义一致。
- **平台适配层** —— 自动屏蔽 shell 差异与路径分隔符差异。

## 📊 可观测性

- **结构化日志** —— 默认输出结构化日志，便于接入 ELK / Loki。
- **执行追踪** —— 记录每条命令的耗时、退出码与上下文。
