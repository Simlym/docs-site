# 架构概览

本页从高层次介绍 OpenClaw 的整体设计、模块划分与数据流，帮助你理解项目全貌。

## 设计目标

OpenClaw 的架构围绕以下目标展开：

1. **单一职责分层** —— 入口、调度、工具、配置各司其职。
2. **可插拔** —— 工具与插件通过统一接口接入，核心不依赖具体实现。
3. **跨平台抽象** —— 平台差异封装在适配层，上层逻辑保持一致。

## 模块划分

```
┌──────────────────────────────────────────────┐
│                   CLI 入口                    │
│            (openclaw 命令、补全)               │
└──────────────────────┬───────────────────────┘
                       │
┌──────────────────────▼───────────────────────┐
│                  命令调度器                    │
│        (参数解析、子命令路由、别名)             │
└──────────────────────┬───────────────────────┘
                       │
        ┌──────────────┼──────────────┐
        │              │              │
┌───────▼──────┐ ┌─────▼──────┐ ┌────▼─────────┐
│  内置工具集  │ │  插件管理  │ │  自定义工具   │
│ (sysinfo 等) │ │ (plugin)   │ │ (tools/ 目录) │
└──────────────┘ └────────────┘ └──────────────┘
        │              │              │
┌───────▼──────────────▼──────────────▼────────┐
│             平台适配层 / 运行时               │
│      (Shell 抽象、路径、日志、配置加载)        │
└──────────────────────────────────────────────┘
```

### 关键模块说明

| 模块 | 职责 |
| --- | --- |
| **CLI 入口** | 解析用户输入，处理 `--help` / `--version`，加载自动补全 |
| **命令调度器** | 将输入映射到具体工具或插件，处理子命令嵌套与别名 |
| **内置工具集** | 随核心发布的常用工具，开箱即用 |
| **插件管理** | 负责插件的安装、更新、卸载与生命周期钩子调用 |
| **自定义工具** | 扫描 `tools/` 目录，将脚本自动注册为子命令 |
| **平台适配层** | 封装 Shell、路径分隔符、日志后端等平台差异 |

## 数据流

以执行 `openclaw sysinfo` 为例：

1. **CLI 入口** 接收原始命令与参数。
2. **命令调度器** 解析出子命令 `sysinfo`，定位到内置工具。
3. **平台适配层** 为该工具注入运行时上下文（日志、配置、当前平台信息）。
4. 工具执行完毕，返回结果与退出码。
5. 调度器统一输出结构化日志。

## 扩展点

- **新增内置工具** —— 实现统一的工具接口并在核心注册。
- **编写插件** —— 通过插件市场分发，遵循插件规范。
- **本地自定义** —— 直接把脚本放入 `~/.openclaw/tools/` 即可被识别。

::: info 小结
OpenClaw 采用「薄核心 + 可插拔扩展」的架构：核心只负责入口与调度，具体能力由内置工具、插件与自定义工具提供。这种设计让 OpenClaw 既轻量，又能灵活适应不同团队的需求。
:::
